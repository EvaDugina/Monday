# Demo — MONDAY

## Паспорт

- **Проект:** MONDAY
- **Стадия:** POC B
- **Версия:** v0.1.3
- **Последнее обновление:** 2026-07-06
- **Кому показываем:** техлид / начальник / внутренний пользователь

---

## Что делает система

MONDAY помогает вести личные и небольшие проектные задачи на одной компактной доске. Пользователь создает карточки, раскладывает их по контекстам, закрывает в архив, синхронизирует состояние с сервером и может локально украсить фон изображениями.

---

## Из чего состоит

- **Web UI** — доска задач, архив, модалки редактирования, поиск, фоновые декорации и статусы синхронизации.
- **API** — Express-сервис для чтения/записи versioned snapshot, auth-контекста и backup-снимков.
- **SQLite** — хранит текущий snapshot доски и последние backup-снимки.
- **Deploy-контур** — Docker Compose для локального запуска и hosted-режимов.

---

## Доменные термины

- **Задача** — карточка с заголовком, описанием, категорией, дедлайном, срочностью и статусом.
- **Категория** — фиксированный контекст доски: `Страсти`, `Бытец`, `Тело`, `Projects`.
- **Архив** — отдельный экран для закрытых задач.
- **Фоновая декорация** — локальное изображение за доской; не синхронизируется с сервером.

---

## Что система может и чего НЕ может

### Может

- Создавать, редактировать, закрывать, восстанавливать и удалять задачи.
- Показывать дедлайны, срочность и синюю метку еженедельного повтора.
- Перемещать задачи между категориями, в том числе drag&drop.
- Сохранять состояние локально и синхронизировать его с сервером.
- Делать ручные и автоматические backup-снимки серверного состояния.
- Добавлять фоновые изображения через upload или drag&drop и показывать parallax.

### НЕ может

- Давать независимые доски каждому пользователю — текущая модель хранит общий snapshot.
- Делать real-time совместное редактирование — конфликт версий решается явно.
- Синхронизировать фоновые изображения между устройствами — это локальная настройка браузера.
- Заменять полноценную PM-систему с ролями, комментариями, чеклистами и аналитикой.

## Как хранятся данные

### Task

```json
{
  "title": "Подготовить демо",
  "category": "projects",
  "deadline": { "kind": "recurring", "mode": "week", "weekday": 1 },
  "urgent": true,
  "status": "open"
}
```

### ServerTasksState

```json
{
  "tasks": ["Task[]"],
  "version": 12,
  "updatedAt": "2026-07-06T10:00:00.000Z"
}
```

### BackgroundDecoration

```json
{
  "name": "sticker.webp",
  "left": 66,
  "top": 10,
  "opacity": 0.2,
  "depth": 0.63
}
```

Связи: сервер хранит только задачи и backup-снимки; фоновые декорации лежат отдельно в `localStorage` текущего браузера.

## Демо-сценарии

### Сценарий 1: Быстро завести и уточнить задачу

**Действие:** создать задачу в категории, открыть карточку, добавить описание, срочность и еженедельный повтор.

**Ожидаемый результат:** задача остается на доске, получает метку `СРОЧНО` или `ЕЖЕНЕДЕЛЬНО`, изменения сохраняются после reload.

### Сценарий 2: Переместить и закрыть задачу

**Действие:** перетащить задачу в другую категорию, отметить checkbox, открыть архив и восстановить задачу.

**Ожидаемый результат:** завершенная задача зачеркивается, скрывается с задержкой, появляется в архиве и может вернуться на активную доску.

### Сценарий 3: Украсить фон

**Действие:** загрузить изображение через кнопку в header или перетащить файл на страницу, затем подвигать указатель и прокрутить доску.

**Ожидаемый результат:** изображение появляется за доской, не мешает карточкам, реагирует parallax-эффектом и удаляется кнопкой очистки.

### Сценарий 4: Проверить сохранность

**Действие:** создать backup, обновить страницу, временно выключить API или открыть вторую вкладку для конфликта.

**Ожидаемый результат:** UI показывает статус синхронизации, не теряет локальные изменения и предлагает загрузить серверную версию при конфликте.

- **Локально:** `http://localhost:8080` после `npm run dev`.
- **Hosted:** см. актуальный deployment path в `README.md` и `.docs/technical.md`.
