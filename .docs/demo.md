# Demo — MONDAY

## Паспорт

- **Проект:** MONDAY
- **Стадия:** POC B
- **Версия:** v0.1.32
- **Последнее обновление:** 2026-07-08
- **Кому показываем:** техлид / начальник / внутренний пользователь

---

## Что делает система

MONDAY помогает вести личные и небольшие проектные задачи на одной компактной доске. Пользователь создает карточки, раскладывает их по настраиваемым контекстам, закрывает в архив, синхронизирует состояние с сервером и может украсить фон изображениями, которые (вместе с выбранным городом погоды) синхронизируются между всеми его браузерами.

---

## Из чего состоит

- **Web UI** — доска задач, архив, модалки редактирования, фоновые декорации и статусы синхронизации.
- **API** — Express-сервис для чтения/записи versioned snapshot, auth-контекста и backup-снимков.
- **SQLite** — хранит текущий snapshot доски и последние backup-снимки.
- **Deploy-контур** — Docker Compose для локального запуска и hosted-режимов.

---

## Доменные термины

- **Задача** — карточка с заголовком, описанием, категорией, дедлайном, срочностью и статусом.
- **Категория** — пользовательский контекст доски с названием и цветом; стартовый набор: `Страсти`, `Бытец`, `Тело`, `Projects`.
- **Архив** — отдельный экран для закрытых задач.
- **Фоновая декорация** — изображение за доской на слое высотой со страницу; его можно добавить drag&drop, выбрать кликом, перемещать, пропорционально масштабировать за углы и удалять в режиме редактирования фона. Байты картинки хранятся на сервере, а её положение — в настройках аккаунта, поэтому фон одинаков во всех браузерах пользователя.

---

## Что система может и чего НЕ может

### Может

- Создавать, редактировать, закрывать, восстанавливать и удалять задачи.
- Открывать модалку редактирования задачи кликом по строке названия и менять название уже созданной задачи в этой модалке.
- Добавлять категории, переименовывать секции кликом по заголовку и менять цвет категории кликом по точке.
- Перемещать категорию со всеми задачами в архив, восстанавливать ее или удалять из архива.
- Показывать дедлайны, срочность, закрепление и повторы как badge в строке задачи.
- Перемещать задачи между категориями, в том числе drag&drop.
- Сохранять состояние локально и синхронизировать его с сервером.
- Делать ручные и автоматические backup-снимки серверного состояния.
- Показывать температуру и локальную Yandex light-иконку погоды в левом верхнем углу вне `app__inner`, выбирать город из стилизованного списка с локальными SVG-флагами, включая Тбилиси, держать `погода live` включённой по умолчанию и менять силу canvas-дождя по прогнозу.
- Добавлять фоновые изображения через drag&drop, управлять ими из floating-панели, выбирать изображение кликом, перемещать, пропорционально масштабировать за углы и удалять их в режиме редактирования фона, показывать насыщенный фон, hover-акцент и прокрутку вместе со страницей.
- Держать изменения фона черновиком до явной кнопки сохранения и удерживать горизонтальное положение изображений относительно центра доски.
- Держать текст задач читаемым поверх фоновых изображений за счет прозрачных liquid glass-подложек категорий.
- Подкрашивать фон доски голубым по ясности прогноза и показывать плывущие облака при облачной/пасмурной погоде.
- Переключать светлую и тёмную тему кнопкой в правом верхнем углу; ночью (23:00–06:00) тёмная тема включается автоматически.

### НЕ может

- Давать независимые доски каждому пользователю — текущая модель хранит общий snapshot (рассчитана на одного пользователя).
- Делать real-time совместное редактирование — конфликт версий решается явно, а фон/город обновляются в других браузерах после перезагрузки, а не мгновенно.
- Добавлять новый фон без сети — загрузка картинки требует доступного сервера (уже добавленный фон работает офлайн из кэша).
- Заменять полноценную PM-систему с ролями, комментариями, чеклистами и аналитикой.

## Как хранятся данные

### Task

```json

  "title": "Подготовить демо",
  "category": "projects",
  "deadline": { "kind": "recurring", "mode": "week", "weekday": 1 },
  "urgent": true,
  "status": "open"
}
```

### ServerTasksState

```json
{
  "categories": [
    { "key": "projects", "label": "Projects", "color": "#7048e8" },
    { "key": "ideas", "label": "Идеи", "color": "#0c8599", "status": "archived", "archivedAt": "2026-07-06T10:30:00.000Z" }
  ],
  "tasks": ["Task[]"],
  "settings": {
    "backgroundDecorations": [
      { "id": "d1", "imageId": "b7f0...", "name": "sticker.webp", "left": -120, "top": 10, "width": 180, "height": 135, "opacity": 0.82, "rotation": -5, "depth": 0.63 }
    ],
    "weatherCityId": "tbilisi"
  },
  "version": 13,
  "updatedAt": "2026-07-06T10:00:00.000Z"
}
```

### BackgroundDecorationRef (внутри settings)

```json
{
  "id": "d1",
  "imageId": "b7f0-...-uuid",
  "name": "sticker.webp",
  "left": -120,
  "top": 10,
  "width": 180,
  "height": 135,
  "opacity": 0.82,
  "rotation": -5,
  "depth": 0.63
}
```

Связи: сервер хранит задачи, категории, настройки аккаунта (`settings` с фоном и городом) и backup-снимки, а также отдаёт same-origin endpoint для текущей погоды Open-Meteo. Байты фоновых изображений лежат в отдельной серверной таблице `background_images` и отдаются через `GET /api/backgrounds/:id`; в `settings` хранится только ссылка `imageId` + геометрия.

## Демо-сценарии

### Сценарий 1: Быстро завести и уточнить задачу

**Действие:** добавить категорию, поменять ей цвет, создать задачу в категории, открыть карточку кликом по строке названия, добавить описание, срочность, закрепление и повтор.

**Ожидаемый результат:** категория сохраняет название и цвет, задача получает цветовой акцент категории, длинное название обрезается с троеточием в строке, название редактируется и сохраняется из модалки, `СРОЧНО` и символ закрепления слева от названия, а повтор или дедлайн отображается badge справа; изменения сохраняются после reload.

### Сценарий 2: Переместить и закрыть задачу

**Действие:** перетащить задачу в другую категорию, отметить checkbox, открыть архив и восстановить задачу.

**Ожидаемый результат:** завершенная задача зачеркивается, скрывается с задержкой, появляется в архиве и может вернуться на активную доску.

### Сценарий 3: Убрать категорию в архив

**Действие:** нажать кнопку архива в заголовке категории, подтвердить перенос, открыть архив, восстановить категорию или удалить ее.

**Ожидаемый результат:** категория исчезает с активной доски вместе с задачами, появляется отдельной карточкой в архиве и удаляется только из архива.

### Сценарий 4: Украсить фон

**Действие:** перетащить изображение на страницу, включить режим редактирования фона через floating-панель, кликнуть фото, увидеть синюю рамку и угловые точки, передвинуть фото, изменить размер за угол, обновить страницу до сохранения, затем повторить и сохранить изменения кнопкой выхода.

**Ожидаемый результат:** изображение появляется за доской без блеклого фильтра, прокручивается вместе со страницей без pointer-parallax, категории остаются читаемыми на прозрачных glass-подложках, черновые изменения до сохранения не переживают reload, после сохранения фото сохраняет новую позицию и пропорциональный размер относительно центра доски, удаляется кнопкой в правом верхнем углу выбранной картинки. Если открыть доску во втором браузере и обновить страницу — тот же фон появляется и там.

### Сценарий 4: Проверить сохранность

**Действие:** создать backup, обновить страницу, открыть вторую вкладку и изменить доску без reload после изменения в первой вкладке, затем временно выключить API.

**Ожидаемый результат:** UI показывает статус синхронизации, при reload берет состояние с сервера, не показывает conflict-баннер во второй вкладке и явно переходит в offline при недоступности API.

## Где посмотреть вживую

- **Локально:** `http://localhost:8080` после `npm run dev`.
- **Hosted:** см. актуальный deployment path в `README.md` и `.docs/technical.md`.
- **Доступ:** dev-режим без auth; hosted-режим требует настроенный auth-контур.
- **Что готово к показу:** сценарии создания задач, архива, backup, server sync/offline и синхронизации фона/города между браузерами.
