# Demo — MONDAY

## Паспорт

- **Проект:** MONDAY
- **Стадия:** POC B
- **Версия:** v0.1.21
- **Последнее обновление:** 2026-07-06
- **Кому показываем:** техлид / начальник / внутренний пользователь

---

## Что делает система

MONDAY помогает вести личные и небольшие проектные задачи на одной компактной доске. Пользователь создает карточки, раскладывает их по настраиваемым контекстам, закрывает в архив, синхронизирует состояние с сервером и может локально украсить фон изображениями.

---

## Из чего состоит

- **Web UI** — доска задач, архив, модалки редактирования, поиск, фоновые декорации и статусы синхронизации.
- **API** — Express-сервис для чтения/записи versioned snapshot, auth-контекста и backup-снимков.
- **SQLite** — хранит текущий snapshot доски и последние backup-снимки.
- **Deploy-контур** — Docker Compose для локального запуска и hosted-режимов.

---

## Доменные термины

- **Задача** — карточка с заголовком, описанием, категорией, дедлайном, срочностью и статусом.
- **Категория** — пользовательский контекст доски с названием и цветом; стартовый набор: `Страсти`, `Бытец`, `Тело`, `Projects`.
- **Архив** — отдельный экран для закрытых задач.
- **Фоновая декорация** — локальное изображение за доской на слое высотой со страницу; его можно добавить drag&drop, перемещать, менять по размеру и удалять в режиме редактирования фона, но оно не синхронизируется с сервером.

---

## Что система может и чего НЕ может

### Может

- Создавать, редактировать, закрывать, восстанавливать и удалять задачи.
- Открывать модалку редактирования задачи кликом по строке названия; само название после создания остается read-only.
- Добавлять категории, переименовывать секции кликом по заголовку и менять цвет категории кликом по точке.
- Перемещать категорию со всеми задачами в архив, восстанавливать ее или удалять из архива.
- Показывать дедлайны, срочность, закрепление и повторы как badge в строке задачи.
- Перемещать задачи между категориями, в том числе drag&drop.
- Сохранять состояние локально и синхронизировать его с сервером.
- Делать ручные и автоматические backup-снимки серверного состояния.
- Показывать температуру и локальную Yandex light-иконку погоды в левом верхнем углу вне `app__inner`, выбирать город из стилизованного списка с Тбилиси и включать лёгкий дождь поверх доски при дождливом прогнозе.
- Добавлять фоновые изображения через drag&drop, управлять ими из floating-панели, перемещать, растягивать за края и удалять их в режиме редактирования фона, показывать насыщенный фон, hover-акцент и прокрутку вместе со страницей.
- Сохранять изменения фона явной кнопкой выхода из режима редактирования и удерживать горизонтальное положение изображений относительно центра доски.
- Держать текст задач читаемым поверх фоновых изображений за счет прозрачных liquid glass-подложек категорий.

### НЕ может

- Давать независимые доски каждому пользователю — текущая модель хранит общий snapshot.
- Делать real-time совместное редактирование — конфликт версий решается явно.
- Синхронизировать фоновые изображения между устройствами — это локальная настройка браузера.
- Синхронизировать выбранный город погоды между устройствами — это локальная настройка браузера.
- Заменять полноценную PM-систему с ролями, комментариями, чеклистами и аналитикой.

  "deadline": { "kind": "recurring", "mode": "week", "weekday": 1 },

### ServerTasksState

```json
{
  "categories": [
    { "key": "projects", "label": "Projects", "color": "#7048e8" },
    { "key": "ideas", "label": "Идеи", "color": "#0c8599", "status": "archived", "archivedAt": "2026-07-06T10:30:00.000Z" }
  ],
  "tasks": ["Task[]"],
  "version": 13,
  "updatedAt": "2026-07-06T10:00:00.000Z"
}
```

### BackgroundDecoration

```json
{
  "anchor": "center",
  "name": "sticker.webp",
  "left": -120,
  "top": 10,
  "opacity": 0.82,
  "depth": 0.63
}
```

Связи: сервер хранит задачи, категории и backup-снимки; фоновые декорации и выбранный город погоды лежат отдельно в `localStorage` текущего браузера.

## Демо-сценарии

### Сценарий 1: Быстро завести и уточнить задачу

**Действие:** добавить категорию, поменять ей цвет, создать задачу в категории, открыть карточку кликом по строке названия, добавить описание, срочность, закрепление и повтор.

**Ожидаемый результат:** категория сохраняет название и цвет, задача получает цветовой акцент категории, длинное название обрезается с троеточием и остается read-only в модалке, `СРОЧНО` и символ закрепления слева от названия, а повтор или дедлайн отображается badge справа; изменения сохраняются после reload.

### Сценарий 2: Переместить и закрыть задачу

**Действие:** перетащить задачу в другую категорию, отметить checkbox, открыть архив и восстановить задачу.

**Ожидаемый результат:** завершенная задача зачеркивается, скрывается с задержкой, появляется в архиве и может вернуться на активную доску.

### Сценарий 3: Убрать категорию в архив

**Действие:** нажать кнопку архива в заголовке категории, подтвердить перенос, открыть архив, восстановить категорию или удалить ее.

**Ожидаемый результат:** категория исчезает с активной доски вместе с задачами, появляется отдельной карточкой в архиве и удаляется только из архива.

### Сценарий 4: Украсить фон

**Действие:** перетащить изображение на страницу, включить режим редактирования фона через floating-панель, передвинуть фото, изменить размер, сохранить изменения кнопкой выхода и удалить лишнее.

**Ожидаемый результат:** изображение появляется за доской без блеклого фильтра, прокручивается вместе со страницей без pointer-parallax, категории остаются читаемыми на прозрачных glass-подложках, фото сохраняет новую позицию и размер относительно центра доски, удаляется кнопкой в правом верхнем углу картинки.

### Сценарий 4: Проверить сохранность

**Действие:** создать backup, обновить страницу, временно выключить API или открыть вторую вкладку для конфликта.

**Ожидаемый результат:** UI показывает статус синхронизации, не теряет локальные изменения и предлагает загрузить серверную версию при конфликте.

## Где посмотреть вживую

- **Локально:** `http://localhost:8080` после `npm run dev`.
